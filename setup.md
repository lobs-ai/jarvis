# Setup

The app code is in the repo, but the **STT/TTS model files are not** — they're
large binaries downloaded or generated locally (git-ignored; see `.gitignore`).
Run the steps below once on a fresh clone before `bin/jarvis start`.

Requires: `pnpm`, `cmake`, `uv` (Python), and a macOS machine for the CoreML/ANE path.

```sh
pnpm install                    # JS workspace deps
```

## STT — whisper.cpp (`sidecars/stt/`)

Driven by the Makefile. Downloads/builds into git-ignored paths.

```sh
make -C sidecars/stt            # clone whisper.cpp + build whisper-server + download model
make -C sidecars/stt coreml     # generate CoreML/ANE encoder (multi-minute, separate venv)
make -C sidecars/stt status     # show what's present / missing
```

Produces (all git-ignored):

| Path | ~Size | Source |
| --- | --- | --- |
| `sidecars/stt/whisper.cpp/` | — | clone of https://github.com/ggml-org/whisper.cpp |
| `sidecars/stt/models/ggml-base.en.bin` | 148 MB | `download-ggml-model.sh base.en` |
| `sidecars/stt/models/ggml-base.en-encoder.mlmodelc/` | 79 MB | generated by `make coreml` |
| `sidecars/stt/.venv-coreml/` | — | temp venv for CoreML generation |

The `coreml` step is what makes the 0.3s transcript latency budget; without it,
whisper falls back to Metal (still works, slower). See the Makefile header.

## TTS — Kokoro + optional Chatterbox (`sidecars/tts/`)

```sh
bash sidecars/tts/setup.sh      # create .venv (Python 3.11), install deps, download Kokoro model
```

Produces (all git-ignored):

| Path | ~Size | Source |
| --- | --- | --- |
| `sidecars/tts/.venv/` | — | Python 3.11 venv (`requirements.txt`) |
| `sidecars/tts/kokoro-v1.0.onnx` | 310 MB | https://github.com/thewh1teagle/kokoro-onnx/releases/download/model-files-v1.0/kokoro-v1.0.onnx |
| `sidecars/tts/voices-v1.0.bin` | 27 MB | same release, `voices-v1.0.bin` |

Kokoro (`:7422`) is the fast local default. Chatterbox (`:7425`, zero-shot voice
cloning, ~30s/sentence) is optional — start it with `sidecars/tts/start-chatterbox.sh`
and drop reference clips into `sidecars/tts/voices/*.wav` (also git-ignored).

## Run

```sh
bin/jarvis start                # sidecars auto-start if their assets exist
bin/jarvis status
```
